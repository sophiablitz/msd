Task 1: Finally or just the code?

Compare the two code fragments. The first one uses finally to execute the code after try..catch:

try {work work}
catch (e) {handle errors}
finally {cleanup the working space}

The second fragment puts the cleaning right after try..catch:

try {work work}
catch (e) {handle errors}
cleanup the working space

We definitely need the cleanup after the work, doesnâ€™t matter if there was an error or not.

Is there an advantage here in using finally or both code fragments are equal?

The difference between using finally versus code after the try...catch is a try...catch...finally will execute the code in the finally block even if the try...catch blocks have return statements. Finally executes before returning from try or catch.

If there is such an advantage, then give an example when it matters.

In the event that cleanup must be done before returning from a function within the try...catch, finally is the place to clean up.
